# AI 内容透明声明规范

[English](./SPEC.en.md)

## 1. 适用范围

本规范定义一套最小化的 AI 参与声明格式，适用于文章、博客、通讯、笔记、文档等以文本为主的发布内容。

## 2. 核心字段

### `ai_ratio`

- 类型：整数
- 范围：`0` 到 `100`
- 含义：AI 对最终发布内容的估算贡献占比
- 说明：这是对最终产出贡献度的自我估算，不等同于投入时长

### `human_ratio`

- 类型：整数
- 范围：`0` 到 `100`
- 含义：人类对最终发布内容的估算贡献占比
- 规则：`ai_ratio + human_ratio = 100`

### `ai_roles`

- 类型：字符串数组
- 含义：由 AI 实际承担的具体任务
- 示例：`"提纲起草"`、`"语言润色"`、`"标题建议"`、`"翻译初稿"`

### `human_roles`

- 类型：字符串数组
- 含义：由人类实际承担的具体任务
- 示例：`"选题确定"`、`"论证设计"`、`"事实复核"`、`"最终审核"`

## 3. 规则要求

1. `ai_ratio` 与 `human_ratio` 之和必须为 `100`。
2. 当 `ai_ratio > 0` 时，`ai_roles` 应至少包含一个明确分工。
3. 当 `human_ratio > 0` 时，`human_roles` 应至少包含一个明确分工。
4. 占比应反映最终发布结果中的贡献，而不只是起草时间或工具使用频率。
5. 分工描述应明确、可理解，避免仅写“辅助”“参与过”等模糊表述。

## 4. 示例 JSON

```json
{
  "version": "1.0",
  "ai_ratio": 35,
  "human_ratio": 65,
  "ai_roles": [
    "提纲起草",
    "语言润色",
    "标题建议"
  ],
  "human_roles": [
    "选题确定",
    "论证设计",
    "事实复核",
    "最终审核"
  ],
  "note": "作者自愿进行的透明声明。"
}
```

## 5. 展示规范

1. 声明必须放在文章、页面或帖文开头。
2. 建议放在正文之前，或标题与基础信息之后、正文开始之前。
3. 声明必须清晰可见、可读，不应藏在脚注、折叠区、弱化样式区域中。
4. 声明不得误导。若 AI 已形成实质性参与，不应仍以“纯人工创作”等说法呈现。
5. 若使用图片卡片展示，建议同步保留文本信息或可访问元数据，避免信息只存在于图片中。
